# Media Dock 3.0 implementation sequence

Each slice must build, pass its relevant automated checks, and run as a real end-to-end path before the next slice expands the product. Work proceeds on an isolated 3.0 line and does not rewrite the 2.1.2 stable generation.

## 1. Isolated foundation

- Establish the 3.0 development line, v3 data namespace, versioned preload contract, test seams, and new product entry.
- Introduce the authoritative Media Task Engine shell, SQLite migration harness, Problem model, and revisioned snapshots.
- Keep 2.1.2 source, data, and published artifacts independently usable.

## 2. First vertical slice: local media

- Inspect a local file through Source Dock.
- Choose a representative Deliverable Recipe and compile an immutable Task Plan.
- Execute a real FFmpeg operation through isolated staging.
- Deliver, verify, and index the result.
- Present the complete path through the adaptive Luminous Workshop shell, Workbench, and task state UI.

## 3. Network media

- Add the network source adapter and a public-link recipe.
- Introduce versioned managed-runtime staging, verification, activation, and rollback for yt-dlp and related tools.
- Drive scoped progress from structured execution events and complete safe delivery.

## 4. Authentication and batches

- Import MediaCookies packages as Authentication Profiles without exposing secrets.
- Add collection inspection, independent Media Tasks inside Task Batches, and resource-aware Scheduling Profiles.
- Verify per-source and per-profile limits and backoff.

## 5. Recovery and management

- Complete Task Center, explicit interrupted-task recovery, staging retention, cancellation, and safe cleanup.
- Complete Deliverable Library relinking and System Center runtime, authentication, storage, and diagnostic operations.
- Export user-reviewed sanitized diagnostic packages.

## 6. Product completeness

- Finish Simplified Chinese and English catalogs, light and dark themes, optional system following, Expert Inspector, keyboard behavior, reduced motion, and compact layouts.
- Reconstruct the selected Media Berth mark into production icon assets with real transparency and platform-safe margins.
- Verify the four product spaces and the one-primary-action Workbench behavior.

## 7. Candidate releases

- Publish opt-in alpha candidates first, then beta and release candidates as evidence grows.
- Build and test the actual macOS and Windows assets, publish checksums, and record every release gate.
- Promote to Stable only when `docs/release/3.0-release-gates.md` is fully satisfied.
