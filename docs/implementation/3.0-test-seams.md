# Media Dock 3.0 test seams

The shared 3.0 architecture confirms the following public seams for TDD. Tests describe behavior through these boundaries and do not query SQLite tables, call private helpers, or mock internal collaborators.

## Media Task Engine

Observe source inspection, Task Plan creation, task commands, workspace snapshots, revision changes, restart recovery, Problems, and Deliverable Records through the engine's public application interface.

## Versioned preload API

Observe command validation, serializable read-only snapshots, revisioned notifications, and rejection of unsupported inputs through the public Media Dock API contract. Do not call individual Electron IPC handlers as the behavior seam.

## Filesystem delivery

Observe destination validation, cross-platform Delivery Names, staging isolation, verified final delivery, collision behavior, and cleanup through task outcomes and files visible in a temporary user-owned destination. Do not inspect private staging implementation unless a user-visible safety invariant requires it.

## Managed runtime operations

Observe staged verification, activation, rollback, operation progress, and pinned task versions through System Operation commands and snapshots. Executable download and process boundaries may use local substitutes; internal manifests are not the assertion surface.

## Renderer journeys

Observe Creator View and the four product spaces through rendered labels, available actions, focus behavior, and the versioned preload substitute. React component state and implementation structure are not test seams.

Each cycle remains vertical: one failing behavior at one confirmed seam, the minimum implementation to pass, then the next behavior. Refactoring waits for the review stage.
