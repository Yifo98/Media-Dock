# Media Dock 3.0 product brief

## Product promise

Media Dock is a local media-processing workspace that turns network or local Sources into dependable Deliverables through understandable Media Tasks. It serves creators first and progressively reveals technical controls without reducing the product to a downloader or command console.

## Product shape

- Workbench creates tasks through Source Dock, Source Inspection, a Deliverable Recipe, and one contextual primary action.
- Task Center owns queueing, recovery, cancellation, batches, and task history.
- Deliverable Library indexes user-owned files without taking custody or silently moving media.
- System Center owns Authentication Profiles, Managed Runtimes, application updates, storage, diagnostics, language, theme, and preferences.
- Expert Inspector reveals task plans, authentication references, runtime versions, generated commands, and bounded raw evidence on demand.

## Experience and identity

Luminous Workshop uses warm light-first surfaces, graphite typography, restrained sea-glass cyan and lilac accents, and a complete dark theme. The selected compact Media Berth mark identifies Media Dock, while MediaCookies keeps its distinct Passage Mark. The adaptive shell supports wide, medium, and compact windows, Windows display scaling, keyboard focus, accessible contrast, and reduced motion without card walls, ambient neon, or decorative skins.

## Architecture

The Electron main process contains one authoritative Media Task Engine. Source adapters inspect without side effects; Deliverable Recipes compile into immutable Task Plans; resource-aware scheduling drives isolated execution and Task Staging Areas; SQLite persists metadata and recovery state; and versioned Managed Runtimes activate only after verification. The renderer submits validated commands and reads revisioned snapshots through one versioned preload API instead of maintaining another queue truth.

## Safety and compatibility

Media Dock 3.0 uses `Media Dock Data/v3/`, imports compatible 2.1 data only by copy, never writes back to the legacy generation, and keeps Cookie values out of databases, logs, snapshots, and diagnostic packages. Delivery uses cross-platform safe names, writable-path validation, explicit replacement, verified finalization, and cleanup rules that cannot delete user Deliverables.

## Delivery strategy

Electron, React, TypeScript, and Vite remain the platform. Implementation proceeds through runnable vertical slices, 2.1.2 remains the stable maintenance generation, and 3.0 advances through opt-in alpha, beta, and release-candidate builds. Stable requires automated checks plus real packaged macOS and Windows evidence.
