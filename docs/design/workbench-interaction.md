# Workbench interaction

## One primary action per phase

Workbench presents one obvious next action for the current creation phase instead of a permanent cluster of equally weighted controls.

| Phase | Primary action | Contextual actions |
| --- | --- | --- |
| No source | Add a link, file, or folder | None |
| Source entered | Inspect source | Clear stays inside Source Dock |
| Source inspected and recipe selected | Start processing | Advanced choices open in Expert Inspector |
| Task active | View task | Cancel belongs to that task |
| Deliverable ready | Use deliverable | Reveal in folder belongs to the deliverable |

`Stop all` is a confirmed dangerous action in Task Center. Retry, cancel, relink, reveal, and other lifecycle actions remain attached to the object they affect and never return as a permanent Workbench button row.
