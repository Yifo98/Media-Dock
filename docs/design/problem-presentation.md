# Problem presentation

## Creator View

Show what needs attention, which task or system capability is affected, and the safest next action. Use the Problem title and summary rather than raw command output. A Problem keeps its stable code visible for support and search.

## Expert Inspector

Reveal the affected stage, timestamps, task and step identifiers, managed-runtime versions, exit status, generated command, and bounded raw output. Sensitive values remain redacted at the source rather than hidden only with CSS.

## Diagnostic package

Before saving, list the categories of information that will be included. Default sanitization excludes Cookie values and files, authentication contents, user-home detail, URL query strings and fragments, unrelated tasks, and unbounded logs. Media Dock never uploads the package; the user owns the saved file and decides whether to share it.
