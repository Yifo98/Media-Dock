## Windows 原生必做

先把仓库中的 `scripts/` 与上述 ZIP、manifest 放到同一份 2.1.2 工作副本，然后执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/verify-windows-package.ps1 `
  -PackagePath "release/2.1.2/Media Dock-2.1.2-win.zip" `
  -YtDlpManifestPath "release/2.1.2/YT-DLP-WINDOWS.json" `
  -ChecksumPath "release/2.1.2/SHA256SUMS.txt" `
  -WriteChecksum
```

该命令必须实际成功运行包内：

- `yt-dlp.exe --version`
- `deno.exe --version`
- `ffmpeg.exe -version`
- `ffprobe.exe -version`

随后在非系统盘、带空格和中文的路径（例如 `F:\媒体测试\Media Dock 2.1.2`）完成：

1. 解压并双击 `Launch Media Dock.bat`。
2. 分别在 Windows 100%、125%、150% 缩放下检查中英文下载面板。
3. 选择一次中文输出目录，再取消一次目录选择，界面均不得崩溃。
4. 打开 cookies 目录和下载目录，确认 Explorer 正常打开。
5. 若旧数据目录中存在损坏的 yt-dlp 或旧版 Deno，同时尝试“修复 yt-dlp”和“更新 Deno”：第二项操作必须被禁用，状态与日志不得在两个工具间来回切换，下载进度日志最多每 10% 一条。
6. 在目标 Windows 的实际系统代理 / PAC / 防火墙环境下完成至少一次工具修复或更新；成功后刷新环境并重启，版本必须可识别且不再提示同一更新。若失败，日志必须包含阶段、目标主机和底层错误，原有可执行文件不得被半下载文件覆盖。
7. 在 yt-dlp 损坏的预检状态下尝试开始下载，界面队列必须保持总数 0、进行中 0；修复成功后再继续下载。
8. 使用体积小、无需登录的公开媒体完成一次下载并导出任务日志。
9. 用包内 ffprobe 验证输出包含有效时长与媒体流，并实际播放一次。
