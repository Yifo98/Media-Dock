# Media Dock

Media Dock 是一个把网络内容与本地媒体转化为可用成品的本地媒体处理工作台。产品围绕媒体任务的完整生命周期组织，而不是围绕单个下载器或工具组织。

## Language

**来源（Source）**：
用户交给 Media Dock 处理的网络链接或本地媒体。
_Avoid_：链接项、输入文件、下载地址

**来源检查（Source Inspection）**：
在不下载媒体、不创建任务且不修改用户文件的前提下识别来源，并给出标题、媒体类型、认证需求、可用成品和风险提示的过程。
_Avoid_：链接解析、预下载、站点探测

**媒体任务（Media Task）**：
从一个来源出发，或从共同生成同一成品的多个来源出发，经过明确处理流程并产生一个或多个成品的独立工作单元。每个媒体任务拥有自己的状态和恢复边界。
_Avoid_：下载任务、工具操作、Job

**任务批次（Task Batch）**：
为共享设置和批量操作而组合的一组独立媒体任务；批次不取代成员任务各自的状态、重试与成品。
_Avoid_：批量任务、播放列表任务、下载队列

**系统活动（System Operation）**：
安装、更新、验证或回滚托管运行工具等不属于媒体任务的后台操作。每项系统活动拥有独立身份、状态、进度和诊断记录。
_Avoid_：后台任务、安装任务、全局状态

**调度策略（Scheduling Profile）**：
控制媒体任务如何共享网络、处理器和磁盘资源的用户选择。创作视图提供稳妥、均衡和快三档策略，底层调度仍遵守站点、认证和系统资源的安全上限。
_Avoid_：并发任务数、线程数、性能模式

**任务中断（Task Interruption）**：
媒体任务因软件关闭、崩溃或执行环境消失而停止但仍保留恢复信息的情况；中断不是任务失败，也不会在应用启动时静默恢复执行。
_Avoid_：失败、暂停、仍在运行

**问题（Problem）**：
阻止或限制来源检查、任务执行、成品使用或系统运行的结构化事实。问题具有稳定编号、类别、发生阶段、用户可理解的摘要、建议操作和脱敏诊断证据。
_Avoid_：报错、异常日志、失败信息

## Task States

**等待中（Queued）**：
媒体任务已经创建并等待获得执行资源。
_Avoid_：待命、未开始、Ready

**处理中（Running）**：
媒体任务正在执行准备、获取、处理或交付阶段。
_Avoid_：运行中、下载中、100%

**需要处理（Needs Attention）**：
媒体任务保留了上下文，但需要用户解决权限、来源、存储、运行环境或中断问题后才能继续。
_Avoid_：失败、异常、错误

**已完成（Completed）**：
媒体任务的处理流程已经结束，所有预期成品均已交付并建立索引。
_Avoid_：下载完成、进度 100%、成功

**已取消（Cancelled）**：
用户明确终止且当前不再等待继续执行的媒体任务。
_Avoid_：停止、暂停、放弃

## Processing Stages

**准备（Preparing）**：验证来源、认证材料、目标位置和处理能力。

**获取（Acquiring）**：从网络或本地来源取得任务所需媒体内容。

**处理（Processing）**：执行合并、转码、提取、字幕或其他媒体变换。

**交付（Delivering）**：完成文件落位、完整性确认和成品索引。

**处理流程（Media Flow）**：
Media Dock 为媒体任务执行的来源识别、内容取得、媒体处理与成品交付过程。
_Avoid_：命令链、下载流程、工具流水线

**成品方案（Deliverable Recipe）**：
用户对期望成品的选择，例如最佳质量视频、兼容性视频、仅音频或视频与字幕包；成品方案描述结果，不要求用户理解底层工具命令。
_Avoid_：下载模式、格式参数、命令预设

**任务计划（Task Plan）**：
从来源检查结果和成品方案编译出的可复现执行描述，固化任务的来源、目标、处理步骤、输出位置和运行工具版本。任务开始后计划不被静默改写。
_Avoid_：当前命令、参数列表、运行配置

**任务暂存区（Task Staging Area）**：
媒体任务独占的受管空间，用于下载片段、中间媒体和待验证成品。暂存内容不是成品，只有完成验证与交付后才对用户显示为可用文件。
_Avoid_：下载目录、缓存文件夹、临时输出

**成品（Deliverable）**：
处理流程完成后交付给用户、可以直接使用的媒体文件或相关文件集合。
_Avoid_：输出文件、结果、下载产物

**交付名称（Delivery Name）**：
任务计划中预先展示并经过目标平台安全化的成品文件名。交付名称保留有意义的 Unicode 内容和稳定辨识信息，默认不覆盖既有文件。
_Avoid_：输出模板、保存名称、文件标题

**成品索引（Deliverable Record）**：
成品库保存的可恢复记录，描述成品、来源、所属任务和用户文件路径，但不拥有或复制实际媒体文件。文件离开原路径后，记录仍然存在并等待重新定位。
_Avoid_：媒体副本、内部文件、缓存记录

**旧版导入（Legacy Import）**：
把 2.1 系列中可识别的偏好、Cookie 包和有效历史记录复制为 3.0 数据的可选过程；旧版导入从不修改、移动或删除原数据。
_Avoid_：数据迁移、原地升级、同步

**托管运行工具（Managed Runtime）**：
Media Dock 为执行媒体任务而维护的 yt-dlp、Deno、FFmpeg 等版本化工具。新版本只有在完成完整性和运行检查后才能成为活动版本，既有版本不会被原地覆盖。
_Avoid_：内核文件、依赖程序、工具副本

**认证配置（Authentication Profile）**：
由用户导入并由 Media Dock 按来源匹配的一组本地认证材料。配置具有站点、名称和健康状态，但 Cookie 等敏感值不进入任务数据库、日志或界面快照。
_Avoid_：Cookie 文件、登录账号、认证目录

**来源入口（Source Dock）**：
工作台中接收链接、本地文件或文件夹并完成来源识别的统一任务入口；成品选择发生在来源识别之后。
_Avoid_：链接输入框、上传区、工具选择器

**创作视图（Creator View）**：
Media Dock 的默认工作视图，只呈现提交来源、选择成品、跟踪进度和使用成品所需的信息。
_Avoid_：基础模式、简易模式、新手模式

**专业检查器（Expert Inspector）**：
按需展示媒体任务的认证材料、运行环境、原始命令、详细日志和高级处理选项的上下文区域。
_Avoid_：高级页面、调试面板、专业模式

## Product Spaces

**工作台（Workbench）**：
创建媒体任务并关注当前处理活动的默认产品空间。
_Avoid_：首页、下载面板、控制台

**任务中心（Task Center）**：
查找、管理和恢复所有媒体任务的产品空间。
_Avoid_：任务列表、队列页面、历史记录

**成品库（Deliverable Library）**：
浏览成品并发起打开、定位或继续处理操作的产品空间。
_Avoid_：最近任务、输出列表、文件浏览器

**系统中心（System Center）**：
管理认证材料、运行环境、软件更新和产品偏好的产品空间。
_Avoid_：配置页、设置面板、启动自检
